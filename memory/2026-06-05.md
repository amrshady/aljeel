### Both-Token Rule — Correct Application
**WHERE to apply:** Name-based resolvers (full_evidence_agent PRIORITY 2,
  emp_registration rule, employee_resolver_v2 L4/L5 name matching)
**WHERE NOT to apply:** L0 (source system emp_no), message/filename emp_no extraction
**Rule:** Both first AND last name tokens must appear in candidate employee name.
  If passenger has only 1 meaningful token, single-token still used.
  Common Arabic names matching alone (ABEER, AMMAR) are false positives.
  Exception: tribal names (AL DIGHRIR vs ALSHAIEB) are data quality issues, not pipeline bugs.
